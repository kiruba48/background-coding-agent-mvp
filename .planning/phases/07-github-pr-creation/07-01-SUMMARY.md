## Key Implementation Decisions

### Branch Name Format
`agent/<slugified-task>-YYYY-MM-DD`
- Slugification: lowercase → non-alphanumeric to hyphen → collapse runs → trim hyphens
- Example: `'maven dependency update'` → `'agent/maven-dependency-update-2026-03-02'`

### PR Body Sections (in order)
1. `## Task` — original task verbatim
2. `## Changes` — agent finalResponse + diffStat in fenced code block (capped at 3000 chars)
3. `## Verification` — pass/fail badges with `<details>` blocks for error output
4. `## LLM Judge` — verdict badge with reasoning in `<details>` block (capped at 2000 chars)
5. `## Breaking Changes` or `## ⚠️ Potential Breaking Changes` — always present
6. `---\n*Generated by background-coding-agent*`

### Breaking Change Heuristics
Three conservative signals:
- `/BREAKING CHANGE/i` — explicit keyword
- `/^-\s*(export\s+(class|function|const|type|interface))/m` — removed exported symbol
- `/major version bump/i` — version increment declaration

### Error Handling Strategy
- **Hard prerequisites** (missing GITHUB_TOKEN, unparseable remote URL): throw immediately before try/catch, so caller knows it's a setup failure
- **Push rejection** (branch already exists on remote): throw inside try/catch → returns `PRResult.error` with branch name included
- **API failures** (PR creation, rate limits): caught → returns `PRResult.error` (non-fatal)

### Test Mock Approach
- `vi.hoisted()` used to declare shared mock functions before `vi.mock` factories execute
- Octokit constructor mock uses `function(this: ...)` (regular function) — arrow functions cannot be used with `new`

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Fixed vi.mock hoisting - arrow function not usable as constructor**
- **Found during:** Task 2 (GREEN phase, first test run)
- **Issue:** Initial test wrote `vi.fn().mockImplementation(() => ...)` for Octokit mock. `vi.mock` factories are hoisted before variable declarations, causing "Cannot access before initialization". Additionally, arrow functions cannot be used with `new Octokit(...)`.
- **Fix:** Used `vi.hoisted()` to declare all mock fns before `vi.mock` factories. Changed Octokit mock to use `function(this: ...)` regular function constructor.
- **Files modified:** `src/orchestrator/pr-creator.test.ts`
- **Commit:** 5005951

**2. [Rule 1 - Bug] Fixed toStartWith/toEndWith not available in vitest**
- **Found during:** Task 2 (GREEN phase, first test run)
- **Issue:** Vitest does not expose `toStartWith`/`toEndWith` matchers. Used them in two `generateBranchName` tests.
- **Fix:** Replaced with `.startsWith()`/`.endsWith()` + `.toBe(true)` assertions.
- **Files modified:** `src/orchestrator/pr-creator.test.ts`
- **Commit:** 5005951
