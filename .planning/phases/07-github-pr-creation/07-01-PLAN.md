**`generateBranchName(taskType: string): string`**
- Lowercase, replace non-alphanumeric with `-`, collapse runs of `-`, trim `-` from start/end
- Append YYYY-MM-DD date suffix: `new Date().toISOString().slice(0, 10)`
- Prefix: `agent/`
- Example: `'maven dependency update'` → `'agent/maven-dependency-update-2026-03-02'`

**`buildPRBody(opts: { task, finalResponse, diffStat, verificationResults, judgeResults, breakingChangeWarnings }): string`**
- Section order (per CONTEXT.md decisions): Task, Changes, Verification, LLM Judge, Breaking Changes
- **Task section**: `opts.task` verbatim
- **Changes section**: `opts.finalResponse` (agent narrative) followed by diff stat in a fenced code block. Cap diffStat at 3000 chars — if longer, append `\n...(truncated)`.
- **Verification section**: For each VerificationResult, show `✅ pass` or `❌ fail` badge next to the verifier type. Use a `<details>` block containing the rawOutput of any errors (label = 'Show output'). If no verificationResults, show 'No verification results recorded.'
- **LLM Judge section**: Show verdict badge (`✅ APPROVE` or `❌ VETO`). Put full reasoning in a `<details>` block labeled 'Show reasoning'. Cap reasoning at 2000 chars — if longer, append `...(truncated)`. If judgeResults is undefined or empty, show 'Judge not run.'
- **Breaking Changes section**: Always present. If breakingChangeWarnings is empty, show `None detected.` If non-empty, show each as a bullet `- <warning>`. Use `⚠️ Potential Breaking Changes` as header when warnings exist, plain `## Breaking Changes` when clean.
- **Footer**: `---\n*Generated by background-coding-agent*`

**`GitHubPRCreator` class:**
```typescript
export class GitHubPRCreator {
  constructor(private workspaceDir: string) {}

  async create(opts: {
    taskType: string;
    originalTask: string;
    retryResult: RetryResult;
    branchOverride?: string;  // from --branch flag
  }): Promise<PRResult>
}
```

`create()` implementation steps:
1. Read `GITHUB_TOKEN` from `process.env`. If not set, throw `new Error('GITHUB_TOKEN environment variable is required for --create-pr')`.
2. Parse owner/repo from `git remote get-url origin` (handle both HTTPS `github.com/owner/repo` and SSH `git@github.com:owner/repo` formats). If parsing fails, throw with the raw remote URL included in the message.
3. Determine branch name: use `opts.branchOverride` if provided, else `generateBranchName(opts.taskType)`.
4. Get diff stat: `git diff --stat HEAD~1 HEAD` in workspaceDir (fall back to `'No changes detected'` on error).
5. Get full diff for breaking change detection: `git diff HEAD~1 HEAD --no-color` (cap to 8000 chars before scanning — don't send to API, just scan locally).
6. Run `detectBreakingChanges(fullDiff)` to get `breakingChangeWarnings`.
7. Build PR body using `buildPRBody(...)`.
8. **Push branch via simple-git** (per CONTEXT.md: if branch already exists on remote, fail with clear error):
   - Construct authed URL: `https://x-access-token:${token}@github.com/${owner}/${repo}.git`
   - CRITICAL: Never log the authed URL (token would leak). If DEBUG logging is on, redact the token.
   - `const git: SimpleGit = simpleGit(this.workspaceDir)`;
   - Check for uncommitted changes: `git status --porcelain`. If output is non-empty, run `git add --all` + `git commit -m 'chore: agent changes'`.
   - Try `await git.checkoutLocalBranch(branchName)`. If it throws (branch exists locally), call `await git.checkout(branchName)` instead.
   - Push: `await git.push(authedUrl, `HEAD:refs/heads/${branchName}`, { '--force-with-lease': null })`.
   - If push fails with "already exists" or "rejected" message: throw with the conflicting branch name included (user must use `--branch` to override or resolve manually).
9. **Create PR via Octokit**:
   - `const octokit = new Octokit({ auth: token })`.
   - Fetch default branch: `const { data: repoData } = await octokit.rest.repos.get({ owner, repo })` → `repoData.default_branch`.
   - Check for existing open PR: `await octokit.rest.pulls.list({ owner, repo, state: 'open', head: `${owner}:${branchName}`, base: baseBranch })`. If found, return `{ url: existingPRs[0].html_url, created: false, branch: branchName }`.
   - Generate PR title: `Agent: ${opts.taskType} ${new Date().toISOString().slice(0, 10)}`.
   - Create PR: `await octokit.rest.pulls.create({ owner, repo, title, body: prBody, head: branchName, base: baseBranch, draft: false })`.
   - Return `{ url: pr.html_url, created: true, branch: branchName }`.
10. Wrap the entire flow in try/catch. On error, return `{ url: '', created: false, branch: branchName, error: err.message }` — caller decides whether to re-throw or log.
