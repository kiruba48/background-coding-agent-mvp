**Create `src/orchestrator/verifier.ts`** with 4 exported functions:

```typescript
import { promisify } from 'node:util';
import { execFile } from 'node:child_process';
import { access } from 'node:fs/promises';
import { join } from 'node:path';
import { VerificationResult, VerificationError } from '../types.js';
import { ErrorSummarizer } from './summarizer.js';
```

**`buildVerifier(workspaceDir: string): Promise<VerificationResult>`:**
1. Pre-check: `access(join(workspaceDir, 'tsconfig.json'))` — if missing, return `{ passed: true, errors: [], durationMs: 0 }` (skip gracefully with a note logged)
2. Run: `execFileAsync('npx', ['tsc', '--noEmit'], { cwd: workspaceDir, timeout: 60_000, maxBuffer: 10 * 1024 * 1024 })`
3. Exit code 0 → `{ passed: true, errors: [], durationMs }`
4. Non-zero → extract errors via `ErrorSummarizer.summarizeBuildErrors(stderr + stdout)`, return `{ passed: false, errors: [{ type: 'build', summary, rawOutput }], durationMs }`

**`testVerifier(workspaceDir: string): Promise<VerificationResult>`:**
1. Pre-check: Look for vitest config. Check for any of: `vitest.config.ts`, `vitest.config.js`, `vitest.config.mts`, `vitest.config.mjs`, or a `vitest` key in `package.json`. If none found, return `{ passed: true, errors: [], durationMs: 0 }` (skip gracefully)
2. Run: `execFileAsync('npx', ['vitest', 'run'], { cwd: workspaceDir, timeout: 120_000, maxBuffer: 10 * 1024 * 1024 })`
3. Exit code 0 → pass. Non-zero → extract via `ErrorSummarizer.summarizeTestFailures(stderr + stdout)`, return fail result

**`lintVerifier(workspaceDir: string): Promise<VerificationResult>`:**
1. Pre-check: Look for ESLint config — check for `eslint.config.mjs`, `eslint.config.js`, `eslint.config.cjs`, `eslint.config.mts`, `eslint.config.cts`. If none found, return passed:true (skip gracefully)
2. **Diff-based lint checking (only new violations):**
   a. Run `execFileAsync('git', ['stash'], { cwd: workspaceDir })` to save changes
   b. Run `execFileAsync('npx', ['eslint', '.', '--format', 'json'], { cwd: workspaceDir, timeout: 60_000, maxBuffer: 10 * 1024 * 1024 })` — capture baseline error count
   c. Run `execFileAsync('git', ['stash', 'pop'], { cwd: workspaceDir })` to restore changes
   d. Run eslint again on the modified workspace — capture new error count
   e. Compare: if new errors > baseline errors, only the delta are "new violations"
   f. If new errors <= baseline, return passed:true (agent didn't introduce new issues)
   g. If new errors > baseline, return passed:false with error summary from the second ESLint run
3. If git stash fails (e.g., no prior commits, no changes), fall back to a simple lint run where any errors fail
4. Use `ErrorSummarizer.summarizeLintErrors()` for the summary

**Important implementation details for ALL verifiers:**
- Use `maxBuffer: 10 * 1024 * 1024` (10MB) to avoid truncation on large test/build output
- Catch `execFileAsync` errors — the error object has `.stdout` and `.stderr` on the error itself when the process exits non-zero
- Measure timing: record `Date.now()` before and after, compute `durationMs`
- Per-verifier error character budget: use 2000 chars per verifier (each verifier's ErrorSummarizer call handles its own truncation)

**`compositeVerifier(workspaceDir: string): Promise<VerificationResult>`:**
1. Run all 3 verifiers in parallel via `Promise.allSettled([buildVerifier(workspaceDir), testVerifier(workspaceDir), lintVerifier(workspaceDir)])`
2. For any rejected promises (verifier crash), convert to a failed VerificationResult with error type 'custom' and the exception message
3. Aggregate: `passed = all 3 passed`, `errors = concatenation of all error arrays` (Build errors first, then Test, then Lint — per locked decision on ordering)
4. Total `durationMs = max of all three` (since they run in parallel)
5. Log each verifier result with timing: `Build: PASS (2.3s), Test: FAIL (8.1s), Lint: PASS (1.2s)` — per locked decision on timing info in logs
