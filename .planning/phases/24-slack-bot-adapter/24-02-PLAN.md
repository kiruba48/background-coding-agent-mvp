    Create `src/slack/index.ts` with:

    1. Module-level state:
       ```typescript
       const threadSessions = new Map<string, ThreadSession>();
       ```

    2. `validateConfig()` function — checks `process.env.SLACK_BOT_TOKEN` and `process.env.SLACK_APP_TOKEN` are set, throws `Error('Missing SLACK_BOT_TOKEN environment variable')` / `Error('Missing SLACK_APP_TOKEN environment variable')` if not.

    3. `handleAppMention(event, client)` — exported for testability:
       - Compute `threadTs = event.thread_ts ?? event.ts`
       - Strip mention: `const text = stripMention(event.text)`
       - If empty text: post "Please include a task description after the mention." to thread and return
       - Create `ThreadSession` with `createSessionState()` as state, new `AbortController()`
       - Store in `threadSessions.set(threadTs, session)`
       - Build `SlackContext` from `{ client, channel: event.channel, threadTs }`
       - Create `ProjectRegistry` instance
       - Call `processSlackMention(text, ctx, session, registry)` — fire-and-forget with `void processSlackMention(...).catch(err => { client.chat.postMessage({ channel, thread_ts: threadTs, text: 'Internal error: ' + (err as Error).message }); }).finally(() => { threadSessions.delete(threadTs); })`

    4. `handleProceedAction(ack, body, client)` — exported for testability:
       - `await ack()` — MUST be first line (3-second Slack deadline)
       - Extract: `channelId = (body as BlockAction).channel?.id`, `confirmMsgTs = (body as BlockAction).message?.ts`, `threadTs = ((body as BlockAction).message as any)?.thread_ts ?? confirmMsgTs`
       - `const session = threadSessions.get(threadTs)`
       - If no session: post "Session expired or already completed." to thread, return
       - Guard double-click: if `!session.pendingConfirm`, post "Already processing." to thread, return
       - Update confirmation message to remove buttons: `await client.chat.update({ channel: channelId, ts: confirmMsgTs, text: 'Confirmed — running...', blocks: [] })`
       - Resolve the deferred confirm: `session.pendingConfirm.resolve(session.intent ?? null)`, then `delete session.pendingConfirm`

    5. `handleCancelAction(ack, body, client)` — exported for testability:
       - `await ack()` — MUST be first line
       - Extract channelId, confirmMsgTs, threadTs same as proceed
       - `const session = threadSessions.get(threadTs)`
       - If no session: return silently
       - Update confirmation message: `await client.chat.update({ channel: channelId, ts: confirmMsgTs, text: 'Cancelled.', blocks: [] })`
       - Resolve with null: `session.pendingConfirm?.resolve(null)`
       - Cleanup: `threadSessions.delete(threadTs)`

    6. `startSlack()` — exported entry point:
       - Call `validateConfig()`
       - Create Bolt `App` with `{ token: process.env.SLACK_BOT_TOKEN, appToken: process.env.SLACK_APP_TOKEN, socketMode: true, logLevel: LogLevel.WARN }`
       - Register: `app.event('app_mention', async ({ event, client }) => handleAppMention(event, client))`
       - Register: `app.action('proceed_task', async ({ ack, body, client }) => handleProceedAction(ack, body, client))`
       - Register: `app.action('cancel_task', async ({ ack, body, client }) => handleCancelAction(ack, body, client))`
       - `await app.start()`
       - Log: `console.log('Slack bot connected via Socket Mode')`

    7. Export `threadSessions` as `getThreadSessions()` getter for testing (or export the Map directly for test access).

    - For the validateConfig tests: set and unset env vars with `vi.stubEnv`
    - For action handlers: construct mock `body` objects matching `BlockAction` shape with `channel.id`, `message.ts`, `message.thread_ts`
