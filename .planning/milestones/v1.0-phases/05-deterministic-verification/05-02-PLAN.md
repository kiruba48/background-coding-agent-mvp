**Create `src/orchestrator/verifier.test.ts`** with vitest tests covering all verifier functions.

Use `vi.mock('node:child_process')` and `vi.mock('node:fs/promises')` to mock subprocess execution and file access. Follow the same mock patterns established in Phase 4 tests (retry.test.ts, summarizer.test.ts).

**Test structure:**

```typescript
import { describe, it, expect, vi, beforeEach } from 'vitest';
```

Mock `execFile` from `node:child_process` (the promisified version) and `access` from `node:fs/promises`.

**buildVerifier tests:**
1. Returns passed:true when tsc exits with code 0
2. Returns passed:false with extracted errors when tsc exits non-zero (mock stderr with TypeScript error format)
3. Returns passed:true (skip) when tsconfig.json is missing (access throws ENOENT)
4. Error summary uses ErrorSummarizer.summarizeBuildErrors format

**testVerifier tests:**
1. Returns passed:true when vitest exits with code 0
2. Returns passed:false with extracted failures when vitest exits non-zero
3. Returns passed:true (skip) when no vitest config found
4. Measures duration (durationMs > 0)

**lintVerifier tests:**
1. Returns passed:true when no new lint violations (baseline == current)
2. Returns passed:false when new lint violations introduced (current > baseline)
3. Returns passed:true (skip) when no eslint config found
4. Falls back to simple lint when git stash fails (no prior commits)
5. Error summary uses ErrorSummarizer.summarizeLintErrors format

**compositeVerifier tests:**
1. Returns passed:true when all 3 verifiers pass
2. Returns passed:false when any verifier fails (errors from all failing verifiers aggregated)
3. Error ordering: Build errors first, then Test, then Lint
4. Handles verifier crash (rejected promise) gracefully — converts to VerificationError
5. Duration is approximately the max of all 3 verifiers (parallel execution)

**Important mock patterns:**
- Mock `execFile` to return `{ stdout, stderr }` for success, or throw error with `.stdout` and `.stderr` properties for failure (mimic child_process behavior where non-zero exit throws)
- Mock `access` to resolve for existing files, reject with `{ code: 'ENOENT' }` for missing files
- Use `vi.clearAllMocks()` in `beforeEach` to reset state between tests
- For lintVerifier diff tests: mock `git stash` and `git stash pop` calls, plus two separate `eslint` calls (baseline and current)

Target: 20-30 tests total covering all branches and edge cases.
