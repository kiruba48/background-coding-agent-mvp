### Integration Defect

**Cancelled task history status (non-critical)**

- **Location:** `src/repl/session.ts` line 137
- **Symptom:** When `runAgent()` returns `{ finalStatus: 'cancelled' }`, history records it as `'failed'` instead of `'cancelled'`
- **Root cause:** Ternary only checks for `'success'`, treats everything else as `'failed'`
- **Fix:** Add `'cancelled'` branch: `result.finalStatus === 'cancelled' ? 'cancelled' : 'failed'`
- **Test gap:** No test for the return-based cancellation path (only throw-based AbortError covered)
- **Impact:** History command shows wrong status for cancelled tasks; follow-up context has incorrect status. Does not affect task execution or core SESS-01 functionality.
- **Affected:** SESS-01
