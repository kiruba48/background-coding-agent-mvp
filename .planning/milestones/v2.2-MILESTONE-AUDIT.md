### Key Wiring Confirmed

- `taskCategory` field travels full pipeline: LLM parser → IntentSchema → ResolvedIntent → AgentOptions → PR branch/title/body
- `description` field flows from raw user input through to `buildGenericPrompt` end-state assertion
- `zero_diff` short-circuits at retry.ts before verifier, judge, or PR creation
- `MIN_DIFF_CHARS` threshold shared between judge.ts and retry.ts — consistent behavior
- `REFACTORING_VERB_GUARD` fires in both one-shot and REPL paths

### Integration Notes

- **Latent coupling:** `retry.ts` calls `compositeVerifier` directly for configOnly path rather than through `retryConfig.verifier`. Custom verifiers would bypass config-only routing. Harmless in production (compositeVerifier is always the verifier) but worth noting for future extensibility.
