## Cross-Phase Integration (16/19 wired)

Integration checker verified 18 cross-phase exports properly connected. Found:

### Wiring Gaps

**1. Slack multi-turn history not populated** (Non-blocking)
- `processSlackMention` in `src/slack/adapter.ts` runs the agent and posts results but never calls `appendHistory` to write back to `session.state.history`
- Impact: Follow-up Slack mentions ("now add tests for that") reach the LLM with empty `<session_history>`, causing reference resolution to fail silently
- Affected requirements: FLLW-03 (partial — REPL works, Slack doesn't), SLCK-02 (partial — history context empty for subsequent mentions)
- Severity: **Non-blocking** — Slack was designed as single-turn in v2.3 (SLCK-08/09/10 deferred to future). Multi-turn is a nice-to-have, not a requirement.

### Dead Code

**2. `buildIntentBlocks`** — exported from `src/slack/blocks.ts`, tested, but never called in production
**3. `buildStatusMessage`** — imported in `src/slack/adapter.ts` but never invoked; all status posts use raw text
